* Introdução sobre Concorrência

Comunicação entre processos ou threads é peça chave

#+latex: \vfill

** Processos

Três questões fundamentais na comunicação entre processos (IPC):

1. Como passar informações de um processo para outro
2. Como garantir que dois processos não interfiram em dados compartilhados
   - =/dev/shm/= com =mmap= em memória
   - Arquivos abertos simultaneamente, via =fork()= ou =fopen()=
   - Recursos como interface de rede, etc
3. Como sequenciar corretamente quando existem dependências

#+latex: \vfill

** Threads

Compartilham o espaço de endereçamento para comunicar
- Variáveis globais
- Regiões de memória alocadas no monte (/heap/)
- Os mesmo itens de processos

* Produtor-Consumidor com espera ativa

Espera ativa (/busy wait/)
- O fio de execução não bloqueia nem cede a CPU
- Testar continuamente uma variável é chamado de espera ocupada (/spinlock/)

#+begin_src C :tangle aula08-pc.c
// gcc -Wall -o pc aula08-pc.c -lpthread
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

#define BUF_SIZE 4
#define N 10

int buffer[BUF_SIZE], in = 0, out = 0, counter = 0;

void *produtor(void *arg) {
    for (int prod = 1; prod <= N; prod++) {
        while (counter == BUF_SIZE);
        buffer[in] = prod;
        in = (in + 1) % BUF_SIZE;
        /* lê, pausa, escreve — janela de corrida exposta deliberadamente */
        int tmp = counter;
        usleep(1);
        counter = tmp + 1;
        printf("[P] produziu %2d | counter=%d\n", prod, counter);
    }
    return NULL;
}

void *consumidor(void *arg) {
    for (int i = 0; i < N; i++) {
        while (counter == 0);
        int cons = buffer[out];
        out = (out + 1) % BUF_SIZE;
        /* lê, pausa, escreve — janela de corrida exposta deliberadamente */
        int tmp = counter;
        usleep(1);
        counter = tmp - 1;
        printf("[C] consumiu %2d | counter=%d\n", cons, counter);
    }
    return NULL;
}

int main(void) {
    pthread_t tp, tc;
    pthread_create(&tp, NULL, produtor,   NULL);
    pthread_create(&tc, NULL, consumidor, NULL);
    pthread_join(tp, NULL);
    pthread_join(tc, NULL);
    printf("\ncounter final (deveria ser 0): %d\n", counter);
    return 0;
}
#+end_src

* Condição de Corrida

Vários processos acessam e manipulam os mesmos dados concorrentemente

- O resultado depende da /ordem específica/ em que o acesso ocorre

#+latex: \vfill\pause

** Exemplo histórico: diretório de spool de impressão

- Dois processos leem a variável =in= (próxima vaga livre) ao mesmo tempo
- Ambos obtêm o mesmo valor e escrevem na mesma posição
- Um dos arquivos é sobrescrito e nunca será impresso

#+latex: \vfill\pause

** A crescente importância dos sistemas multicore aumenta o problema

- Mais núcleos significam mais execução paralela real
- Condições de corrida tornam-se mais frequentes e difíceis de depurar

** Requisitos para uma solução correta

1. Exclusão mútua \to apenas um processo na região crítica por vez
2. Progresso \to se ninguém está na região crítica, quem está na entrada
   é candidato e não se pode adiar esta decisão
3. Espera limitada \to há um limite para quantas vezes outros processos
   entram antes de mim

* #1 Desabilitando interrupções (HW)

Processo desabilita interrupções ao entrar na região crítica

- Sem interrupção de relógio, não há chaveamento de contexto
- Processo examina e atualiza a memória compartilhada sem interferência

#+latex: \vfill

** Problemas dessa abordagem

- Dar aos processos de usuário o poder de desligar interrupções é perigoso
  - Se um processo nunca reabilitar, o sistema pode travar
- Em multiprocessadores, afeta somente a CPU que executou a instrução
  - Outras CPUs continuam executando e acessando a memória compartilhada

#+latex: \vfill\pause

** Conclusão

- Técnica útil dentro do próprio kernel (por poucas instruções)
- Não é apropriada como mecanismo geral para processos de usuário

* #2 Variáveis do tipo trava (SW)

# #+ATTR_LATEX: :options fontsize=\normalsize
#+begin_src C :tangle lock_falho.c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>
int lock = 0; // variável compartilhada lock, inicialmente em zero
void *fun(void *arg) {
    while (lock != 0); // espera ativa
    lock = 1;
    printf("Thread %d entrou\n", *(int*)arg);
    usleep(100000); // a região crítica
    printf("Thread %d saiu\n", *(int*)arg);
    lock = 0;
    return NULL;
}
int main(void) {
    pthread_t t[2];
    int id[] = {1, 2};
    for (int i = 0; i < 2; i++) pthread_create(&t[i], NULL, fun, &id[i]);
    for (int i = 0; i < 2; i++) pthread_join(t[i], NULL);
}
#+end_src

** Contém a mesma falha fatal do diretório de spool
- Dois processos estão na região crítica ao mesmo tempo
** Verificar a trava duas vezes antes de armazená-la não ajuda
- A corrida pode ocorrer entre a segunda verificação e o armazenamento

* #4 Algoritmo de Peterson (SW)

Combina alternância com variáveis de interesse (=flag[]=)

#+begin_src C
#define N 2 /* numero de processos */
int turn;                         /* de quem e a vez? */
int interested[N];                /* todos os valores 0 (FALSE) */
void enter_region (int process) { /* processo e 0 ou 1 */
  int other;                      /* processo e 0 ou 1 */
  other = 1 - process;            /* o oposto do processo */
  interested[process] = TRUE;     /* mostra que voce esta interessado */
  turn = process;                 /* altera o valor de turn */
  while (turn == process && interested[other] == TRUE); /* espera ativa */
}

void leave_region(int process) /* processo: quem esta saindo */
{
  interested[process] = FALSE; /* indica a saida da regiao critica */
}
#+end_src

#+latex: \vfill

- =flag[i] = true=: processo Pi declara interesse em entrar
- =turn = j=: cede a vez ao outro processo por cortesia
- Se ambos tentam ao mesmo tempo, o último a escrever =turn= espera

* #4 Algoritmo de Peterson: garantias (SW)

** Satisfaz os três requisitos da seção crítica
1. Exclusão mútua
   - Se ambos na seção crítica, =interested[0] = interested[1] = true=
   - Mas =turn= só pode ser 0 ou 1, não ambos
   - Logo, um deles ficou bloqueado no =while= (contradição)
2. Progresso
   - Se Pj não quer entrar, =interested[j] = false= e Pi entra imediatamente
   - Se ambos querem entrar, =turn= decide; não há adiamento indefinido
3. Espera limitada
   - Quando Pj sai, faz =interested[j] = false= e Pi entra
   - Pi espera no máximo uma entrada de Pj

#+latex: \pause\vfill

** Problemas

- Assume consistência sequencial de memória
  - CPUs modernas tem reordenamento de instruções
  - Caches independentes por núcleo
  - Compilador ou o hardware podem reordenar as escritas
- Não escala além de dois processos na formulação original

* #5 A instrução TSL (HW)

Soluções de software não garantem funcionamento em arquiteturas modernas

- Instruções =load= e =store= podem ser reordenadas pelo hardware

#+latex: \vfill

Instrução =TSL RX,LOCK= (/Test and Set Lock/):

- Lê o conteúdo de =LOCK= para o registrador =RX=
- Armazena um valor diferente de zero em =LOCK=
- Operação atômica e indivisível (barramento de memória travado)

#+latex: \vfill\pause

** Reflexão
- Travar o barramento de memória impede acesso de outras CPUs
- Diferente de desabilitar interrupções (que só afeta uma CPU)

#+latex: \vfill\pause

** Instruções parecidas com TSL
- =XCHG=: troca atômica de registrador e memória
- =CMPXCHG= - /Compare-and-swap/: base dos mutexes e estruturas lock-free
- =XADD= - Incremento atômico: usado em contadores, semáforos
- =BTS/BTR= - /Test-and-set/reset:/ em bits individuais

* Exclusão mútua sem espera ativa

Peterson, TSL e XCHG funcionam mas têm o defeito da espera ativa

- Processo fica em /spinlock/ consumindo tempo de CPU inutilmente
- Ainda que em algumas situações isso possa ser benéfico (esperas curtas)

#+latex: \vfill

** Problema da inversão de prioridade

- Processo H (alta prioridade) e L (baixa prioridade)
- L está na região crítica e H fica pronto para executar
- H faz espera ocupada, mas L nunca é escalonado para sair da região
- Resultado: ambos ficam bloqueados indefinidamente

#+latex: \vfill\pause

** Solução
- Usar primitivas que bloqueiem o processo ao invés de esperar ativamente

  int item;
  while (TRUE) {
    item = produce_item();

*** Consumidor
#+ATTR_LATEX: :options fontsize=\small
#+begin_src C


void consumer(void) {
  int item;
  while (TRUE) {
    if (count == 0)
      sleep();
    item = remove_item();
    count = count - 1;
    if (count == N - 1)
      wakeup(producer);
    consume_item(item);
  }
}
#+end_src

* Dormir e Acordar (Problema)

1. Consumidor lê =count = 0= mas é interrompido antes de dormir
   - Produtor insere item, incrementa =count=, chama =wakeup(consumidor)=
   - Consumidor não está dormindo ainda; o sinal de despertar é perdido @@latex: \pause@@
2. Consumidor retoma e dorme; produtor eventualmente enche o buffer e dorme @@latex: \pause@@
3. Ambos dormem para sempre
